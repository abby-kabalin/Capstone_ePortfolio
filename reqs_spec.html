Requirements Specification

Based on our analysis of user feedback, we have developed a comprehensive list of prioritized requirements for our system. These requirements are categorized into functional, data, software, and hardware components. To ensure clarity and effective implementation, we have assigned priority levels ranging from 1 to 5, where 1 represents the highest priority and 5 represents the lowest priority.


Functional Requirements
Category Description Priority
Extract Users should be able to download a PDF of the data. 1
Extract Users should be able to download a PNG of the data. 1
Extract Users should be able to download a PPT file of the data. 1
Extract Users should be able to download a basic infographic of all data. 1
Query Users should be able to sort data with dropdown menus. 1
Query Users should be able to sort data by county. 1
View Users should be able to update data after selection. 1
Use Users should be able to generate a bar chart with selected data. 1
Use Users should be able to generate a line plot with selected data. 1
Use Users should be able to generate a scatter plot with selected data. 1
Trace Users should be able to use a breadcrumbs hyperlinked path to allow for a trace through user's route to their location 2
View Users should be able to use toggles within the dashboard to filter data. 2
Query Users should be able to search though data. 3
Query Users should be able to compare and contrast data amoungst counties. 3
View Users should be able to see resident attitude data displayed on dashboard. 3
View Users should be able to see description of what data is being displayed, for example county name, community size, region, etc. 3
View Users should be able to display data via time series. 4
Use Users should be able to use an interactable banner as a home for all major toggles, filters and search. 4
View Users should be able to customize visualization colors. 5
Use Users should be able to walk-through a tutorial on first visit to dashboard that walks them through basic functionalities of the dashboard. 5
