Systems Modeling

To model our system beyond the aspects of the user interface, we developed a sequence diagram and a dialog map.

Sequence Diagram

We chose a sequence diagram because this diagram represents the flow of operations and interactions between different aspects of the dashboard.. It provides a dynamic view of the system by illustrating how each function operates with one another and in what order. The sequence diagram helped us visually understand how the system will interact with each user.

  • A. Landing Page - Displays the various counties and provides navigation to the About Page and our Main Visualization Page
  • B. Visualization Page - Shows the users the data theyâ€™ve selected for their chosen county and how they can filter their chosen data
  • C. Database Retrieval - Demonstrates how the user will create a query which will call to the database for the results and return them to the dashboard.
  • D. Export Page - Users have the ability to export their selected data from the dashboard to their chosen file-type.

Dialog Map

We chose a Dialog Map Model to represent our system. A dialog map provides a high-level view of a proposed or actual user interface, showing the various display elements and possible navigation pathways between them (Wasserman 1985; Wiegers 1996). Dialog maps are often used in the realm of software engineering to help organize requirements and visualize decisions a user can make when using the system.

  • A. Landing Page - Provides a hub for all users to find out more information about the project, select various counties, and navigate to the main visual page.
  • B. Main Visual Page - Demonstrates how the users makes selection on what data they want to see and then the dashboards state transitions to display the appropriate visual on the main page.
  • C. Print Report/Visual - Provides a way for the user to export and save the visuals that they have displayed on the dashboard.
